Introduction

Basic architecture

The main aim of the chat bot is to identify the tag of pattern. As soon as we get the respective tag we search for the responses under it and select a response randomly and print it in the output.

    The steps involved in building the chat bot would be something like this:

  1. Collecting the data for tag, patterns and responses
  2. Making a key to convert between strings and binary form (0's and 1's)
  3. Making a relation list between pattern and tag, this will be our training data
  4. Building the neural network model
  5. Training the data
  6. Making an application where we can talk to our chat bot

Setting up intents

Creating intents from visual studio code

This is how we create the intents.json file:

  1. We create a dictionary and make a key called intents.
    {"intents" : }
  2. The value of this key will be a list.
    {"intents" : [] }
  3. Inside this list we will add a dictionary again.
    {"intents" : [ {} ] }
  4. This dictionary will contain tag, patterns and responses.
    {"intents" : [ { "tag" : , "patterns" : , "responses" : } ] }
  5. Then we assign a value to the tag and make a list for patterns and responses
    {"intents" : [ { "tag" : "greetings", "patterns" : [], "responses" : [] } ] }
  6. Now we add a multiple questions inside the patterns and similarly add multiple answers in responses.
    {"intents" : [ { "tag" : "greetings", "patterns" : ["how are you", "hey there", "hello", "good morning"], "responses" : ["Hello! How can I help you today?", "Hey! Nice to meet you, how can I help you?"] } ] }
  7. Now we make another dictionary and repeat the whole process again. At the end our intents file should look something like this:
    {"intents": [ { "tag" : "greetings", "patterns" : ["hey there!", "hello", "how are you", "Whats up", "How are you doing?", "good evening"], "responses" : ["Hello, how can I help you today?", "Hey there, good to meet you! how can I help you today?"] }, { "tag" : "coffee", "patterns" : ["What type of coffee do you sell?", "what are the types of coffee that you make?"], "responses" : ["We have Cappuccino, Latte, Arabica and Iced coffee."] }, { "tag" : "order", "patterns" : ["how can i place order?", "how do i place an order for my coffee?"], "responses" : ["You can visit our website www.coffee.com/placeOrder and place your order online."] }, { "tag" : "services", "patterns" : ["What can you do for me?", "What all things can you do?", "how can you help me?"], "responses" : ["I can tell you the type of coffees we serve and how you can place an order."] }, { "tag" : "good bye", "patterns" : ["bye", "see you later", "thank you for helping me"], "responses" : ["Bye!", "Good bye!"] } ]}

Tokenization

What is tokenization?

Tokenization is the process of breaking down a piece of text into smaller units called tokens. These tokens can be words, phrases, symbols, or other meaningful elements. In simpler terms, it's like chopping a sentence into individual pieces.

Example:
Consider the sentence: "The cat is sleeping."
Tokenization of this sentence would result in the following tokens:

  1. "The"
  2. "cat"
  3. "is"
  4. "sleeping"
Each word in the sentence becomes a separate token. Tokenization is a fundamental step in natural language processing (NLP) and text analysis, allowing computers to understand and work with the individual elements of a piece of text.

Tokenization

What is Word Net Lemmatizer?

WordNet Lemmatizer is a linguistic tool provided by the NLTK (Natural Language Toolkit) library in Python. It is based on WordNet, a lexical database of the English language. The main purpose of a lemmatizer is to reduce words to their base or root form, known as a lemma. In the case of the WordNet Lemmatizer, it maps words to their corresponding lemmas based on their lexical entries in WordNet.

For example:

- The lemma of "running" is "run."
- The lemma of "better" is "good."

WordNet Lemmatizer is particularly useful in natural language processing tasks where having words in their base form simplifies analysis, comparison, and understanding of text data. It's commonly used in tasks like text mining, sentiment analysis, and information retrieval.

Training data

What is bag of words?

The "Bag of Words" (BoW) is a popular technique used in natural language processing (NLP) and information retrieval for representing text data. It simplifies and converts a piece of text into a numerical format that can be understood by machine learning algorithms.

In a Bag of Words model:

The term "Bag of Words" implies that the order of the words is discarded, and the focus is solely on their occurrence in the document. It treats each document as an unordered set of words, creating a "bag" without considering the sequence of words.

Example:

Consider the two sentences:
1. "The cat in the hat."
2. "The cat sat on the mat."

The vocabulary is {"The", "cat", "in", "the", "hat", "sat", "on", "mat"}.

The Bag of Words representation for these sentences might be:
1. [1, 1, 1, 1, 1, 0, 0, 0]
2. [1, 1, 0, 1, 0, 1, 1, 1]

Each element of the vector corresponds to the count of the respective word in the sentence. The order of the words is disregarded, and only their presence is considered. The resulting vectors can be used for various NLP tasks, such as text classification, sentiment analysis, and information retrieval.

  2. We will loop through each word inside the key and append 1 inside the bag if the word is present inside our sentence, otherwise we append 0 and return the bag.
    def convert_to_bag(sentence, key):

     bag.append(1) if word in sentence else bag.append(0)

What is sequential?

tf.keras.Sequential is a high-level API in TensorFlow for building and training neural network models. It allows you to define a linear stack of layers, specifying the architecture of your neural network in a sequential manner. Each layer in the stack has connections only to the previous and next layers, forming a simple feedforward neural network.

Neural network

What is dense layer?

A dense layer, also known as a fully connected layer, is a fundamental building block in neural networks. It is a type of layer where each neuron or node in the layer is connected to every neuron in the previous layer. This connectivity pattern results in a densely connected network of neurons.

Working of dense layer

Imagine you have a bunch of input values. A dense layer takes all these input values, does some math with them (like multiplying and adding), and then applies a rule (activation function) to decide whether each neuron in the layer should be activated or not. Each neuron's decision is based on the input it receives and some internal parameters (weights and biases). The result is a transformed set of values that can capture patterns in the input data. So, a dense layer helps the neural network learn and understand the features in the data.

What is reLU?

ReLU stands for Rectified Linear Unit, and it is an activation function commonly used in neural networks. The function is defined as:
f(x) = max(x,0)

The ReLU activation function returns the input value if it is positive, and zero otherwise. Graphically, this function looks like a straight line for positive values and is flat (zero) for negative values.

What is adam optimizer?

The key features of the Adam optimizer include:

Adaptive Learning Rates: Adam dynamically adjusts the learning rates for each parameter based on their past gradients. It uses two moving averages: the first-order momentum (like in momentum optimization) and the second-order momentum (like in RMSprop). These moving averages are used to adaptively scale the learning rates for each parameter.

Bias Correction: Adam includes a bias correction mechanism to account for the fact that the moving averages are initialized with zeros. This correction helps in the early iterations of training when the estimates of the moments may be inaccurate.

Combination of Momentum and RMSprop: Adam combines the concepts of momentum optimization and RMSprop. The first-order momentum helps the optimization process by accumulating gradients, while the second-order momentum helps in scaling the learning rates based on the historical gradient information for each parameter.
